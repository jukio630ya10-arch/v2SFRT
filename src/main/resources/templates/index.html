¿Qué hace cada página?

El sistema está pensado para la gestión integral de un Almacén Multiservicios: un negocio donde coexisten productos de diferentes categorías, entradas y salidas constantes, y múltiples usuarios operando. Cada página cumple un rol específico dentro del flujo operativo diario.

Página de Productos

Desde esta vista se visualiza el catálogo activo del almacén: nombre, precio, stock y acciones rápidas. Es el núcleo de consulta para saber qué artículos están disponibles en tiempo real y cuáles requieren reposición.

El formulario de productos permite registrar nuevos ítems o editar existentes, asegurando que la data de inventario se mantenga limpia y usable para movimientos y ventas posteriores.

Página de Categorías

Las categorías organizan el catálogo en familias lógicas (por ejemplo: ferretería, limpieza, oficina, alimentos, repuestos, etc.). Esto ayuda a buscar productos más rápido y a ordenar reportes por líneas de negocio.

Desde su formulario se crean y actualizan categorías, evitando duplicidad y mejorando el control de inventario en un entorno multiservicios donde hay variedad de rubros.

Página de Movimientos

Aquí se registran las ENTRADAS y SALIDAS de inventario. Cada movimiento impacta directamente en el stock del producto relacionado, dejando trazabilidad de fecha, cantidad y usuario responsable.

Esta página es clave para auditoría operativa: permite entender por qué subió o bajó el stock y mantener consistencia entre inventario físico y sistema.

Página de Ventas

La vista de ventas lista solo movimientos de tipo SALIDA, simulando el flujo comercial del almacén. Además muestra precio unitario y total por línea para facilitar seguimiento de ingresos estimados por operación.

Esta pantalla ayuda al encargado a revisar rápidamente qué productos se están vendiendo más y qué usuario registró cada salida.

Abrir ventas

Página de Entradas

La vista de entradas lista solo movimientos de tipo ENTRADA, representando el flujo de abastecimiento y reposición del almacén. Además, muestra precio unitario y total por línea para facilitar el seguimiento del valor invertido en la reposición de productos.

Esta pantalla ayuda al encargado a revisar rápidamente qué productos están siendo reabastecidos con mayor frecuencia y qué usuario registró cada entrada.

Página de Usuarios / Vendedores

Este módulo administra el personal que opera el sistema (vendedores, encargados o administradores). Permite registrar usuarios con rol y mantener control de quién ejecuta movimientos de inventario.

En un Almacén Multiservicios, donde varias personas pueden registrar operaciones en simultáneo, esta trazabilidad por usuario es fundamental para seguridad, responsabilidad y análisis de desempeño.

Flujo recomendado de uso

  1. Registrar categorías para estructurar el catálogo.
  2. Registrar productos con precio y stock inicial.
  3. Crear usuarios que operarán el almacén.
  4. Registrar movimientos de entrada/salida según operación diaria.
  5. Monitorear la vista de ventas para revisar salidas valorizadas (precio y total).
